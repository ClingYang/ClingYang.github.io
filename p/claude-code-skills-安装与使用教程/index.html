Claude Code Skills 安装与使用教程

Claude Code Skills 安装与使用教程

2026-05-15
阅读时长: 2 分钟

前言

最近切换到国内模型后发现 Claude Code 的 Skills 市场一直转圈加载不出来，显示 0 个技能。研究了一下发现是因为在线仓库请求被墙了，后来找到了手动安装的方法，顺便记录一下 superpowers 插件的使用方式。

手动安装 Skills（绕过在线市场）

1. 找到 Skills 本地目录

在 Claude Code 里输入：

1

/skills list

看输出里有没有显示 skills 的本地路径。通常是：

  • Windows：C:\Users\你的用户名\.claude\skills\
  • macOS/Linux：~/.claude/skills/

如果 /skills list 显示的是空列表，直接手动创建这个目录就行。

2. 解压 ZIP 到 skills 目录

把下载好的 ZIP 分别解压，确保最终结构是：

1
2
3
4
5

.claude/skills/
├── project-context/
│   └── SKILL.md
└── brainstorming/
    └── SKILL.md

注意：每个 ZIP 里已经包含了文件夹层（如 project-context/SKILL.md），直接解压到 skills/ 目录下即可，不要多套一层文件夹。

3. 刷新验证 

1

/skills refresh

然后验证：

1

/skills list

应该能看到：

  • project-context
  • brainstorming

为什么在线仓库里搜不到

CC Switch -> Skills 管理 -> 仓库，这是在线插件市场，需要从 GitHub 拉取技能列表。切了国内模型后，这个请求大概率被墙或重定向失败，所以一直转圈、显示 0 个技能。

本地 skills 目录和在线仓库是两回事：

  • 在线仓库 = App Store，需要联网下载
  • 本地 skills = 你已经装好的 App，不需要联网

手动放到本地目录后，Claude Code 直接从本地加载，完全不需要走 CC Switch 的仓库界面。

superpowers 插件安装

当 Claude Code 找到 superpowers 插件后，会弹出一个安装确认界面，让你选择安装范围：

选项 范围 说明
为你安装（用户范围） user scope 推荐 全局安装，所有项目都能用
为仓库协作者安装（项目范围） project scope 写进项目配置，团队共享
仅在此仓库为你安装（本地范围） local scope 只在这个项目里用

推荐选第 1 个（user scope），因为 superpowers 是通用开发工作流插件，所有项目都用得上，全局装一次以后自动生效。

装好后输入 /help 应该能看到 /brainstorm 等命令。

“locked by plugin” 是什么意思

看到技能列表里显示 locked 是正常的，不是报错。这些技能被 superpowers 插件统一管理，你不能单独开关某一个。插件启用时全部启用，禁用时全部禁用。锁图标表示"插件托管"。

装完后列表大概长这样：

技能 状态
superpowers:brainstorming on, locked
superpowers:writing-plans on, locked
superpowers:executing-plans on, locked
project-context on, user

注意：superpowers 已经自带了 brainstorm 功能，不需要再额外装 brainstorming.zip 了，功能会重复。

模式 B：手动触发

如果只想快速讨论方案，不想立即写代码：

1

/brainstorm 外扩 W25Q64 存 LVGL 字库的最佳方案

project-context 怎么用（全自动）

完全自动，什么都不用做。它的工作流：

时机 它做什么
你打开项目 自动读 readme.md，告诉你项目当前状态
Claude 改完代码 自动追加变更日志到 readme.md
你说"保存进度" 自动写工作总结到 readme.md
项目结构变了 自动更新技术栈和外设列表

常用指令：

  • "保存进度" / "今天先到这" -> 触发工作总结
  • "查看变更记录" -> 读 readme.md 的变更日志
  • "查看踩坑" -> 读踩坑记录

怎么增加/修改你的需求

Skill 就是 Markdown 文件，直接改 SKILL.md 就行。

两个文件位置：

1
2
3

.claude/skills/
├── project-context/SKILL.md    <- 改这个
└── superpowers/                <- 插件管理的，不建议手动改

需求 1：记录更多东西（比如寄存器配置细节）

打开 project-context/SKILL.md，找到"文档模板"部分，加一节：

1
2
3
4
5
6
7

## 寄存器配置快照
<!-- 关键外设的配置值，方便调试对比 -->

| 外设 | 寄存器 | 当前值 | 备注 |
|------|--------|--------|------|
| SPI1 | SPI_CR1 | 0x034C | 主模式, 9MHz, CPOL=0 |
| TIM2 | TIM_ARR | 7199 | 20kHz PWM |

然后在"代码修改后协议"里加一条规则：如果修改了外设初始化代码，提取关键寄存器值更新到 ## 寄存器配置快照。

需求 2：自动记录每次对话的完整内容

在"任务总结协议"的格式里，把 完成内容 下面的条目改详细：

1
2
3
4
5

**完成内容**：
- `src/pid.c`：新增自适应 PID 函数 `pid_adaptive_update()`
  - 关键算法：用升温斜率查表，表存在 `flash_lookup_table[]`
  - 入口参数：target_temp, current_temp, heat_up_rate
  - 返回值：新的 PWM 占空比

需求 3：换个文件名

全局替换 readme.md -> PROJECT_LOG.md，包括文件位置说明、读取目标、文档模板里的标题。

怎么验证修改生效

改完 SKILL.md 后：

1

/skills refresh

然后测试：打开你的项目，说"帮我写个简单的 GPIO 翻转测试"，看 Claude 是否按新加的规则记录。

总结

  • 在线 Skills 仓库被墙 -> 手动解压到 .claude/skills/ 目录
  • superpowers 插件推荐全局安装（user scope）
  • Brainstorm 自动触发，project-context 全自动运行
  • 自定义需求直接改 SKILL.md，改完 /skills refresh 生效
